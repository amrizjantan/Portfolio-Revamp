MyFlix-Angular

Overview

MyFlix is a single page, responsive movie database web application which includes several interface views that will handle data through the previously-defined REST API endpoints. After registering, users log-in and will gain access to information about movies, directors, and genres. Users may edit their profile and create a list of favorite movies.

  • Role: Developer and Designer
  • Technology: MEAN stack (MongoDB, Express, Angular and Node.js)
  • Skills: Project management
  • Duration: 2 months



    • Purpose & Context

    The goal was to utilize the previously built REST API and familiarise with Angular as a framework, to implement it's unique functionality with a variety of built-in modules and services. Another goal was to practice project management and collaboration where a detailed and straightforward handoffs for other developers were created.



    Objective

    I wanted to build a functioning full-stack web application that showcases my ability to use the MEAN stack. Therefore i built both server-side and client-side for My-Flix from scratch. I was also aware of the importance to establish the suitable aesthetic and design themes for the project. Hence this is an ambitious project to represent full-stack development for my portfolio.



    Approach

    Server-Side

    Using Node.js and Express, I first developed a RESTful API interacting with MongoDB, a non-relational database. The API is accessible through the HTTP methods GET, PUT, POST and DELETE. The classic CRUD methods allow retrieving data from and storing data in the database. Movie information is provided in JSON format. Basic HTTP authentication and JWT authentication allow for user authentication and authorization. Postman was used for testing purposes.



    Client-Side

    The API serves as the foundation for the client-side application. I started with building a welcome page that allows users to register or log in. Since this project is a single page application (SPA), users will be directed to the movie view right after logging in. There are no tabs or reloading of another page. There are several interfaces including views specifically for movie, genre and director information. Users have an own profile view where they may edit their information or delete their profile. They also have a list of favorite movies which they can add to or delete movies from.



    Challenges

    To create an API and accordingly build two different client-side applications with two different library/frameworks i.e. React and Angular. Another challenge was to set up the and maintain the project's environment as well as debugging along the way.



    Retrospective

    The project was designed and completed over a period of two months, 2 weeks longer than planned. Setting up the environment for both API and Web Application was suprisingly more complicated than expected as the technologies used evolving rapidly. The production process was challenging due to new technologies used as well debugging unexpected number of bugs. I wasn't hesitate to reach out for support and advise from other developers once stuck or needed to solve complex issue in order to keep up the progress and meet deadline. This was the leasson I learned from this project. For future similar project I would set a more realistic timeframe knowing the complexity involved. The project met all its objective and has potential for design improvement and more features can be added as well.
